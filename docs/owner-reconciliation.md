# Reconciling owners and pets

## User type
Accountant

## District
Customers Service

## Purpose
Explains how owner and pet data can be reconciled before accounting reports are generated.

## What this file contains
- Every pet should have one valid owner.
- Every billable visit should point to a known pet.
- Owner records should not be duplicated.
- Missing contact fields should be flagged.

## Agent behavior
Open this file when the accountant asks about reconciliation, owner matching, pet ownership, or missing customer data.

## Demo trigger phrases
- reconcile owners
- owner matching
- pet ownership
- missing customer data
