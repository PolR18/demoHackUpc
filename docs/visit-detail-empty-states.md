# Visit detail empty states

## User type
Designer

## District
Visits Service

## Purpose
Defines empty states and fallback UI for visit-related screens.

## What this file contains
- No visits found → Clear filters.
- Visit data unavailable → Retry.
- No billing-related visits → View all visits.

## Agent behavior
Open this file when the designer asks about empty states, error states, fallback screens, or missing visit data.
