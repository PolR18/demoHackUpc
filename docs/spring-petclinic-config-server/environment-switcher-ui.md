# Environment switcher UI

## User type
Designer

## District
Config Server

## Purpose
Defines a UI pattern for switching between local, staging, and production.

## What this file contains
- Current environment must be visible.
- Dangerous environments should be clearly labelled.
- Users should understand when data belongs to staging or production.

## Agent behavior
Open this file when the designer asks about configuration, environments, staging, production, or settings UI.

- environment switcher
- staging
- production
- settings UI
